# Session Handoff 中文说明

这份文档用来解决一个很实际的问题:

如果一轮任务没做完, 下一次会话或者下一个 agent 怎么快速接手?

## 为什么要有 handoff

如果没有 handoff, 常见情况会变成:
- 下个会话重新读大量上下文
- 不知道前一轮做到哪
- 不知道已经排除了哪些路径
- 不知道下一步最该做什么

所以 handoff 的目的就是:

用一份短文档把“当前状态”交给下一轮。

## 推荐 handoff 内容

一个好的 handoff 至少应该包含:
- 当前任务
- 已完成内容
- 未完成内容
- 当前阻塞
- 已验证内容
- 剩余风险
- 下一步建议

## 什么时候写 handoff

最适合这些情况:
- 任务没做完
- 会话太长准备切新会话
- 准备交给别的 agent
- 排查类任务已收敛一部分结论

## 推荐原则

- 短
- 准
- 面向接手者

不要写成长篇流水账。
