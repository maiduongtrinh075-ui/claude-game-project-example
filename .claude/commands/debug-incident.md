# Debug Incident Command

Use this command when debugging an intermittent issue or crash.

## Steps
1. Identify the owning area.
2. Capture logs, versions, and current scope.
3. Load the matching debugging skill or workflow.
4. Narrow the likely root causes.
5. Propose telemetry or the next smallest validation step.

## Output
- suspected root causes
- missing telemetry
- next debugging step
