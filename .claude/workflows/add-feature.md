# Add Feature Workflow

## Use When
Use this workflow when adding a new feature to an existing module.

## Steps
1. Identify the owning module.
2. Read the relevant system doc and module guide.
3. Define the smallest feature slice that can ship.
4. Keep changes local when possible.
5. Avoid broad refactors unless explicitly requested.
6. Update docs if behavior changes in a lasting way.
7. Verify the feature works and does not break existing flow.

## Output
- feature added
- affected module
- unchanged contracts
- follow-up recommendations
