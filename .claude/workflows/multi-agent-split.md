# Multi-Agent Split Workflow

## Use When
Use this workflow when a task can be safely split into separate write scopes.

## Steps
1. Confirm the task is large enough to justify parallel work.
2. Split work by module or file area, not by vague intent.
3. Assign one owner per write scope.
4. Keep one main agent responsible for integration.
5. Avoid parallel edits to the same files.
6. Merge and verify final behavior centrally.

## Output
- split plan
- owner per scope
- integration owner
- risks to watch during merge
