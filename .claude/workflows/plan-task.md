# Plan Task Workflow

## Use When
Use this workflow when a request is too broad and needs to become a one-round task file.

## Steps
1. Identify the owning module.
2. Reduce the request to one goal.
3. Mark what must stay stable.
4. Define allowed scope.
5. Write clear acceptance criteria.
6. Save the result under `tasks/in-progress/`.

## Output
- task file path
- module owner
- allowed scope
- acceptance criteria
