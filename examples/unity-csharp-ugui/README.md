# Unity C# UGUI Example

This example shows how the Claude-oriented project structure can map onto a practical Unity + C# + UGUI game project.

## What This Example Demonstrates
- how root-level Claude docs stay outside the Unity `Assets/` folder
- how gameplay, UI, data, and integration map to Unity script folders
- how one module can have its own `CLAUDE.md`
- how a concrete task can drive one round of expansion

## Start Here
- Read `CLAUDE.md`
- Read `docs/guide.zh-CN.md`
- Read `docs/unity.zh-CN.md`
- Open `Assets/Scripts/Gameplay/Combat/CLAUDE.md`
- Open `tasks/in-progress/combat-ui-phase-1.md`
