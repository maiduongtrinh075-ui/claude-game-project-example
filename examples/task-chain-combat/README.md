# Combat Task Chain Example

This example shows how one module can evolve across multiple task rounds instead of being rewritten in one pass.

## What This Example Demonstrates
- one module expanded in phases
- task files as round-by-round contracts
- when to update docs
- when to record decisions
- how review notes and verification should evolve
